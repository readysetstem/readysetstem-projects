VARIABLES & ASSIGNMENTS

Variables are one of the most important concepts in programming, and you will certainly be using them in every useful program you write. Variables are like labels, or names, for the numbers and values in your programs. Variables are used for keeping track of all the facts and figures you use throughout your code.

Variables can be given any name you choose (although there are some rules to that!) Choosing good variable names makes it easier to write your code, to change your code later and to help others understand your code.

Variable names

Choosing good variable names can make a program easier to read an understand. A variable can generally be any name you want, as long as it contains only:

  • Uppercase (A through Z) or lowercase (a through z) letters

  • Digits (0 through 9)

  • The underscore character ("_")

And, it can't start with a digit, either.

Even though you can name variables any way you want, it's common to write variable names that are longer than one word by separating the words with underscores. And, that's the way we'll generally do it throughout this guide. One exception: for variables that don't change values throughout the program, we'll generally use ALL_CAPS for those variable names.

Other Variable Types

In this tutorial, we've discussed two types of variable values: numbers and strings. But there are many other types of data values that can be used as well:

  • Lists — Lists are ordered sequences of other values. We'll discuss them more in and will be using them frequently in future projects.

  • Booleans - Booleans are a funny name for the values True and False. We'll discuss them more in and you'll experience the concept of booleans lots of different places.

  • None — None is a special value which strangely indicates that there is no value at all.

Those are the most common types of values, and they cover the ones you'll see in the core set of Ready Set STEM projects. But there are more, including types you can even define yourself. By the way, in Python, as in some other programming languages, all values are called objects, and we'll discuss objects more throughout this guide.
